黄粱一梦

正因是梦，尤需真活。

本来是想刷大佬自制OpenWrt来更好做mosdns和文件共享结果光OpenWrt就刷了一天

2022.12.04

最近这玩意儿写的越来越水了，本来是不想写的，但是过程实在是太曲折了，我自己都觉得离谱，一个简单的openwrt都刷了大几个小时，实在受不了不得不写一个

这个是为了加mosdns的时候能有交互界面，我一开始的时候刷的就是高版本的breed，刷的也是原版的openwrt，不知道他们是怎么有的不同的包的交互界面，干脆就重新刷一个大佬做的openwrt固件，遂需从breed重新开始刷

问题

开启ssh失败

第一台

后来刷了有四十分钟才发现，中途还换过u盘，原来是用breed刷回官方固件的时候，ssh不用再次开启，麻透了

第二台

刷了半天才发现，我忘了一件事情，就是不同路由器固件是不一样的，要重新绑定小米账号，重新下载刷写

ssh可以使用，但是无法使用sftp

我在网上找到了教程$^{[3]}$，但是照着做到解压data.tar.gz的时候还是报错了

...:read-only file system

懒得想解决办法，直接换了一台路由器，和楼下人换的，因为我一度以为是之前刷写过高版本breed的原因，结果后来又刷错了

虽然最后发现其实也不需要解决，直接wget就好，官方网站是https，还下不下来，最后只好从我电脑上的ftp服务器下载

上传固件报错

I/O failure while writing temporary file

至今是个玄学，应该是储存不够的原因，但是自定义的时候，加的软件包无论多少也是120~130M，反而是某一次加的不多不少的时候只有70M左右，至今不知道为什么

一直处于恢复模式

无论刷写多少次，重启多少次都是

系统在恢复 (initramfs) 模式下运行

或者说有的时候好像刷成了，但是找不到openwrt的登陆后台了

最后发现是之前配置参数的时候，删除参数和增加环境变量的时候

没点保存！！！

小失误

breed版本

在第二次刷写breed的时候没有确认版本，又重新把新版本刷回去了，md，受不了了

结果到现在一个包都没用上，光忙活openwrt就花了这么久，心累了。，昨天的qqbot都没忙完呢
